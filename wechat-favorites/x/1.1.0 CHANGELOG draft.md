## [1.2.0] - 2026-04-24

### Added / 新增

- **二级分类** — 一级分类下的细分标签，基于关键词正则匹配
  - `classify_favorites.py` 新增 `SUBCATEGORIES` 字典（6 个一级分类 × 共 57 个二级标签）
  - 输出到 `subcategory` 列（管道分隔，可多个）
  - 生物医药（14）、AI科技（9）、投资金融（6）、科学研究（6）、商业财经（5）、生活方式（7）
  - 媒体资讯、政治国际暂无二级标签
  - `--no-subcat` 参数：不输出二级分类

  **Sub-category Classification** — Fine-grained tags under each primary category

- **跨领域标签** — 文章同时涉及两个领域时自动标注
  - `classify_favorites.py` 新增 `CROSS_DOMAIN_RULES`（6 个跨领域标签）
  - 输出到 `cross_domain` 列（管道分隔，可多个）
  - 生物医药+投资(2974)、AI+医疗(546)、AI+投资(339)、生物医药+AI(203)、商业+政治(68)、科学+政策(19)
  - `--no-cross` 参数：不输出跨领域标签

  **Cross-domain Tags** — Auto-label articles spanning two domains

- **LLM prompt 升级** — LLM 分类结果现在包含 `subcategories` 和 `cross_domain` 字段
  - `llm_classify.py` 新增 `SUBCATEGORY_TAGS` 和 `CROSS_DOMAIN_TAGS` 定义
  - Prompt 增加二级分类和跨领域标签选项
  - 响应解析增加 `subcategories` 和 `cross_domain` 过滤

- **normalize_categories.py 升级** — 支持二级分类和跨领域标签的标准化
  - `SUBCATEGORY_KEYS`、`SUBCATEGORY_ALIASES`：二级标签别名映射
  - `CROSS_DOMAIN_KEYS`、`CROSS_DOMAIN_ALIASES`：跨领域标签别名映射
  - 模糊匹配：检查是否是某个 key 的子串

### Changed / 变更

### Added / 新增

- **LLM 二次分类** — 对关键词分类低置信度条目用大模型智能重判
  - `scripts/llm_classify.py`：LLM 分类核心模块，OpenAI 兼容 API
  - `scripts/llm_incremental.py`：增量 LLM 分类，断点续传（`llm_checkpoint.json`）
  - `scripts/merge_llm_results.py`：合并 LLM 检查点结果到 CSV
  - 支持任意 OpenAI 兼容 API（OpenRouter、DeepSeek、OpenAI、本地 vLLM）
  - 环境变量配置：`LLM_API_KEY`（必填）、`LLM_API_URL`、`LLM_MODEL`、`LLM_CONCURRENCY`、`LLM_BATCH_SIZE`、`LLM_TEMPERATURE`
  - 无硬编码密钥，所有敏感信息通过环境变量传入

  **LLM Secondary Classification** — Re-classify low-confidence items with LLM
  - Core module: `llm_classify.py`, OpenAI-compatible API
  - Incremental processing with checkpoint: `llm_incremental.py`
  - Merge checkpoint results: `merge_llm_results.py`
  - Supports any OpenAI-compatible API (OpenRouter, DeepSeek, OpenAI, local vLLM)
  - Environment variable config, no hardcoded secrets

- **分类标签标准化** — 将混合格式标签统一为 8 个标准分类
  - `scripts/normalize_categories.py`：中文/英文/复合标签 → 标准 key
  - `classify_favorites.py --normalize`：集成模式一键标准化
  - `tech` 合并到 `ai` 分类
  - 支持复合标签拆分（`+`、`|`、`,` 分隔）

  **Category Normalization** — Unify mixed-format tags to 8 standard categories
  - Chinese/English/composite tags → standard English key
  - `tech` merged into `ai`
  - Composite tag splitting (`+`, `|`, `,` separators)

- **classify_favorites.py 增强**
  - `--llm`：启用 LLM 二次分类
  - `--llm-only`：全量 LLM 分类（跳过关键词）
  - `--llm-threshold`：自定义触发阈值（默认 0.4）
  - `--normalize`：分类后标准化标签
  - `--input` / `--output`：自定义输入输出路径

- **CLI & Environment Variable Support** — Restored command-line arguments and environment variables for `import_ima.py`
  - `--csv`, `--kb-id`, `--client-id`, `--api-key`, `--state`, `--log`, `--batch-size`
  - Environment variables: `IMA_CLIENT_ID`, `IMA_API_KEY`, `IMA_KB_ID`
  - Priority: CLI args > Environment variables > config.json > ~/.config/ima/

  **命令行与环境变量支持** — 恢复 `import_ima.py` 的命令行参数和环境变量配置

- **Quick Validation Script** — Added `scripts/quick_validate.py` to check dependencies and configuration

  **快速验证脚本** — 新增环境检查

- **LICENSE.txt** — Added MIT License file

- **.gitignore** — Prevent sensitive data from being committed

### Fixed / 修复

- **import_ima.py batch counting** — Unified batch counter, removed duplicate `batch_count` variable
- **import_ima.py rate-limit state** — Now saves state before sleep, preventing progress loss
- **classify_favorites.py cross-platform** — Replaced hardcoded Windows path separator with `os.path.join()`
- **llm_classify.py security** — Removed hardcoded API key, `LLM_API_KEY` environment variable now required

### Changed / 变更

| 4 | ~/.config/ | `client_id`, `api_key` files |
